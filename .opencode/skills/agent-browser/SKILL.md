# Browser Verification for qai

Visual verification skill for frontend agents. Captures UI screenshots in mobile + desktop viewports for comparison.

## Quick Start

### For Public Pages (login, sandbox)
```bash
# 1. Navigate directly (no auth needed)
agent-browser open http://localhost:3000/login
agent-browser wait --load networkidle
agent-browser screenshot artifacts/screenshot.png

# 2. Analyze with look_at tool
```

### For Protected Pages (dashboard, draft, etc.)
```bash
# 1. Use persistent profile (login once, reuse)
agent-browser --profile ~/.qai-browser-profile open http://localhost:3000/dashboard
agent-browser wait --load networkidle
agent-browser screenshot artifacts/screenshot.png

# 2. If not logged in yet, complete login flow first
# 3. Analyze with look_at tool
```

## Prerequisites

Before running browser verification:

1. **Dev server running with MSW mocks**:
   ```bash
   ENABLE_MSW=true pnpm dev
   ```

2. **Auth state file exists** (generated by Playwright):
   ```bash
   test -f tests/e2e/.auth/user.json || pnpm test:e2e
   ```
   If missing or expired, run `pnpm test:e2e` to regenerate.

3. **agent-browser installed**:
   ```bash
   which agent-browser || npm install -g agent-browser
   ```

## Auth State Loading Protocol

qai uses Supabase auth. For **public pages** (login, sandbox), no auth is needed. For **protected pages** (dashboard, draft, etc.), use one of these approaches:

### Option 1: Public Pages (No Auth Needed)

For testing public pages like login or sandbox:
```bash
agent-browser open http://localhost:3000/login
agent-browser open http://localhost:3000/draft-sandbox
# No auth required - these work directly
```

### Option 2: Manual Login (Recommended for Protected Pages)

Log in once via agent-browser and save the state for reuse:
```bash
# Close any existing session
agent-browser close

# Navigate to login
agent-browser open http://localhost:3000/login
agent-browser wait --load networkidle
agent-browser snapshot -i

# Fill login form (using refs from snapshot)
# Example: textbox "Email" [ref=e1], button "Send Magic Link" [ref=e2]
agent-browser fill @e1 "e2e-test@quantasy.test"
agent-browser click @e2

# Complete login flow (check email for magic link or use password if configured)
# Then save the authenticated state:
agent-browser state save artifacts/agent-browser-auth.json

# Future sessions: load this saved state
agent-browser state load artifacts/agent-browser-auth.json
```

### Option 3: Persistent Profile (Best for Repeated Use)

Use a persistent profile directory to maintain login across sessions:
```bash
# First time: login manually with profile
agent-browser --profile ~/.qai-browser-profile open http://localhost:3000/login
# ... complete login ...

# Future sessions: profile remembers auth
agent-browser --profile ~/.qai-browser-profile open http://localhost:3000/dashboard
```

### Known Limitation

The `state load` command with Playwright's `tests/e2e/.auth/user.json` has limited compatibility. The Supabase cookie format requires httpOnly flags that can't be set via agent-browser's cookie commands. Use Option 2 or 3 above for authenticated pages.

**Test user details:**
- Email: `e2e-test@quantasy.test`
- Password: `test-password-123!` (if password login enabled)
- Test league ID: `987654321`

## Dual Viewport Capture Protocol

qai is mobile-first. Always capture BOTH viewports for UI verification:

| Viewport | Dimensions | Device |
|----------|------------|--------|
| Mobile | 375 x 812 | iPhone 13 |
| Desktop | 1440 x 900 | Desktop Chrome |

### Standard Capture Workflow

```bash
# Setup
RUN_ID="task-N-feature-$(date +%Y%m%d-%H%M%S)"
mkdir -p "artifacts/agent-browser/$RUN_ID"

# For public pages (no auth needed):
# agent-browser open "http://localhost:3000/login"

# For protected pages (use persistent profile):
# agent-browser --profile ~/.qai-browser-profile open "http://localhost:3000/dashboard"

# Mobile capture (iPhone 13)
agent-browser set viewport 375 812
agent-browser open "http://localhost:3000/your-page"
agent-browser wait --load networkidle
agent-browser screenshot "artifacts/agent-browser/$RUN_ID/mobile.png"

# Desktop capture
agent-browser set viewport 1440 900
agent-browser open "http://localhost:3000/your-page"
agent-browser wait --load networkidle
agent-browser screenshot "artifacts/agent-browser/$RUN_ID/desktop.png"

# Optional: Save interactive element snapshot
agent-browser snapshot -i > "artifacts/agent-browser/$RUN_ID/elements.txt"
```

### Full Page Screenshots

For long scrolling pages:

```bash
agent-browser screenshot --full "artifacts/agent-browser/$RUN_ID/mobile-full.png"
```

## Artifact Organization

All verification artifacts go in `artifacts/agent-browser/`:

```
artifacts/agent-browser/
└── task-1-login-20260130-143022/
    ├── mobile.png           # Mobile viewport (375x812)
    ├── desktop.png          # Desktop viewport (1440x900)
    ├── mobile-full.png      # Full page mobile (optional)
    ├── desktop-full.png     # Full page desktop (optional)
    └── elements.txt         # Interactive elements snapshot (optional)
```

**Naming convention:** `task-N-feature-YYYYMMDD-HHMMSS`

**Important:** This directory is gitignored. Artifacts are for local verification only.

## MSW Mock Integration

qai uses MSW to mock the Sleeper API for consistent test data.

**Required:** Dev server MUST be started with MSW enabled:
```bash
ENABLE_MSW=true pnpm dev
```

**What MSW mocks:**
- Sleeper API responses (players, leagues, rosters, matchups)
- Test league ID: `987654321`
- Consistent player projections and stats

**Verify MSW is active:**
```bash
agent-browser open http://localhost:3000/dashboard
agent-browser snapshot -i | grep "League"  # Should show mock league data
```

## Verification Workflow

Complete workflow for frontend agents:

```
1. MAKE CHANGES      Code your UI changes
        ↓
2. START SERVER      ENABLE_MSW=true pnpm dev
        ↓
3. AUTH (if needed)  Use --profile for protected pages, or skip for public pages
        ↓
4. DUAL CAPTURE      Mobile (375x812) + Desktop (1440x900)
        ↓
5. ANALYZE           Use look_at tool on screenshots
        ↓
6. ITERATE           Fix issues, re-capture until correct
        ↓
7. CLEANUP           agent-browser close
```

### Example: Verifying a Button Style Change

```bash
# Setup
RUN_ID="button-style-$(date +%Y%m%d-%H%M%S)"
mkdir -p "artifacts/agent-browser/$RUN_ID"

# Navigate (use --profile for protected pages, or test on public sandbox)
agent-browser open http://localhost:3000/draft-sandbox

# Wait for page and find button
agent-browser wait --load networkidle
agent-browser snapshot -i  # Find the button ref

# Capture button state
agent-browser set viewport 375 812
agent-browser screenshot "artifacts/agent-browser/$RUN_ID/mobile.png"

agent-browser set viewport 1440 900
agent-browser screenshot "artifacts/agent-browser/$RUN_ID/desktop.png"

# Now use look_at to analyze:
# - Is button visible?
# - Does styling match requirements?
# - Responsive at both viewports?
```

## Session Isolation (Parallel Agents)

For parallel verification (multiple agents):

```bash
# Agent 1 verifies login page
agent-browser --session verify-login open http://localhost:3000/login

# Agent 2 verifies dashboard (different session)
agent-browser --session verify-dashboard open http://localhost:3000/dashboard

# List active sessions
agent-browser session list

# Each session is isolated:
# - Separate browser instance
# - Separate cookies/storage
# - Independent navigation
```

## Common Verification Patterns

### Page Load Verification

```bash
agent-browser open http://localhost:3000/page
agent-browser wait --load networkidle
agent-browser snapshot -i  # Check expected elements exist
agent-browser is visible "[data-testid=main-content]"
```

### Form Interaction

```bash
agent-browser snapshot -i
# Output: textbox "Search" [ref=e1], button "Submit" [ref=e2]

agent-browser fill @e1 "test query"
agent-browser click @e2
agent-browser wait --load networkidle
agent-browser screenshot form-result.png
```

### Animation/Transition Capture

```bash
# Capture before state
agent-browser screenshot before.png

# Trigger animation
agent-browser click @e1

# Wait briefly for animation
agent-browser wait 500

# Capture after state
agent-browser screenshot after.png
```

### Responsive Layout Validation

```bash
# Test at breakpoints
for viewport in "375 812" "768 1024" "1440 900"; do
  agent-browser set viewport $viewport
  agent-browser screenshot "layout-${viewport// /x}.png"
done
```

## Commands Reference

### Navigation
```bash
agent-browser open <url>          # Navigate
agent-browser back/forward        # History
agent-browser reload              # Refresh
```

### Capture
```bash
agent-browser screenshot [path]   # Screenshot (--full for full page)
agent-browser snapshot -i         # Interactive elements with refs
agent-browser pdf output.pdf      # Save as PDF
```

### Interaction
```bash
agent-browser click @e1           # Click by ref
agent-browser fill @e1 "text"     # Fill input
agent-browser type @e1 "text"     # Type without clear
agent-browser hover @e1           # Hover
agent-browser press Enter         # Key press
agent-browser scroll down 500     # Scroll
```

### State
```bash
agent-browser get text @e1        # Get element text
agent-browser get url             # Current URL
agent-browser is visible @e1      # Visibility check
```

### Settings
```bash
agent-browser set viewport 375 812    # Mobile
agent-browser set viewport 1440 900   # Desktop
agent-browser set device "iPhone 14"  # Device emulation
```

### Session
```bash
agent-browser state save auth.json    # Save auth state
agent-browser state load auth.json    # Load auth state
agent-browser close                   # Close browser
```

## Troubleshooting

### Auth Not Working / Redirects to Login

**Symptom:** Protected pages redirect to /login

**Cause:** Playwright's storage state format has limited compatibility with agent-browser.

**Fix - Use Persistent Profile:**
```bash
# Close existing session
agent-browser close

# Login manually with persistent profile
agent-browser --profile ~/.qai-browser-profile open http://localhost:3000/login
agent-browser snapshot -i  # Find login form refs
agent-browser fill @e1 "e2e-test@quantasy.test"
# Complete login flow...

# Now protected pages work with the profile
agent-browser --profile ~/.qai-browser-profile open http://localhost:3000/dashboard
agent-browser get url  # Should show /dashboard
```

**Alternative - Test Public Pages Only:**
```bash
# Public pages don't need auth
agent-browser open http://localhost:3000/login
agent-browser open http://localhost:3000/draft-sandbox
```

### No Elements Found in Snapshot

**Symptom:** `snapshot -i` returns empty or minimal elements

**Fix:**
```bash
# Wait for page to fully load
agent-browser wait --load networkidle
agent-browser wait 1000  # Additional wait if needed

# Check page has content
agent-browser get text body

# Try full snapshot (not just interactive)
agent-browser snapshot
```

### MSW Not Working

**Symptom:** Real API calls or missing mock data

**Fix:**
```bash
# Ensure dev server started with MSW
# WRONG: pnpm dev
# RIGHT: ENABLE_MSW=true pnpm dev

# Restart dev server with correct env
pkill -f "next dev"
ENABLE_MSW=true pnpm dev
```

### Browser Won't Start

**Symptom:** `agent-browser` commands hang or fail

**Fix:**
```bash
# Install browser binaries
agent-browser install

# On Linux, also install system deps
agent-browser install --with-deps

# Check for stale sessions
agent-browser session list
agent-browser close  # Clean up
```

### Screenshots Are Blank or Wrong Size

**Symptom:** Empty images or incorrect dimensions

**Fix:**
```bash
# Explicitly set viewport before screenshot
agent-browser set viewport 375 812
agent-browser wait 500  # Wait for resize
agent-browser screenshot test.png

# Use --full for full page
agent-browser screenshot --full fullpage.png
```

## Integration with look_at

After capturing screenshots, use the `look_at` tool to analyze them:

```
look_at(
  file_path="artifacts/agent-browser/task-1/mobile.png",
  goal="Verify the dashboard shows the user's leagues and the navigation is visible"
)
```

**Good analysis prompts:**
- "Verify the button is styled with the new primary color"
- "Check if the layout is responsive and content doesn't overflow"
- "Confirm the loading state shows a skeleton, not a blank page"
- "Verify the error message is displayed prominently"
